3. **개발 연결 사고**: 기획 문서를 읽고 개발자가 실제로 구현할 수 있는 결과물을 만드는가

## 중요한 안내

**디자인 방향**

`brief/03-app-screenshots/`의 기존 SEDA 앱 스크린샷은 앱의 기능 구조를 이해하기 위해 제공됩니다. **기존 시각 스타일을 그대로 적용할 필요는 없습니다.** 다만 SEDA가 가진 학습 앱으로서의 신뢰감, 다국가 사용자 맥락, 모바일 네이티브 구현 가능성은 고려해주세요.

**AI 도구**

Claude, Gemini, Codex(ChatGPT), Stitch, Figma 등 어떤 AI 도구든 자유롭게 사용하세요. **도구 비용은 본인 부담입니다.**

AI 활용은 필수입니다. 단, 평가는 특정 도구를 썼는지가 아니라 AI에게 어떤 맥락을 제공했고, 어떤 결과를 채택·폐기했으며, 최종 판단을 어떻게 본인이 책임졌는지에 집중합니다. AI를 어떻게 활용했는지 반드시 `submission/template.md`에 기록해주세요.

**개발 핸드오프**

개발자가 이 디자인을 어떻게 구현할지도 고려해주세요. 핸드오프 형식은 여러분이 정합니다.

**작업 기록**

결과물만으로는 알기 어려운 **작업 과정**을 이해하기 위한 자료입니다. 어떤 방향을 먼저 시도했고, 무엇을 버렸으며, 어떤 판단으로 최종 결과에 이르렀는지가 결과물만큼 중요합니다.

기록 방식은 작업 환경에 따라 다릅니다.

**이 폴더에서 직접 작업하는 경우** (HTML, 로컬 파일, Claude Code 등)

아래 명령어를 실행하면 파일 변경이 `logs/work-events.jsonl`에 자동으로 기록됩니다. 작업이 끝날 때까지 터미널을 닫지 마세요.

```bash
npm run start-assignment
```

기록기는 어떤 파일이 언제 저장되었는지만 기록합니다. 디자인을 평가하거나 개선안을 제안하지 않습니다.

**Stitch, Figma(Figma Make), v0 등 외부 도구로 작업하는 경우**

클라우드 기반 도구는 이 폴더 안의 파일을 직접 변경하지 않아 자동 기록이 되지 않습니다. `recorder/work-log-template.md`를 열어 작업 흐름을 간략히 기록해주세요.

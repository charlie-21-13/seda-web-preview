# Work Recorder

이 기록기는 후보자의 디자인을 평가하거나 개선안을 제안하지 않습니다.

역할은 `workspace/` 폴더 안에서 어떤 작업 파일이 변경되었는지 감지하고, 변경 내용을 `logs/work-events.jsonl`에 순서대로 기록하는 것입니다.

## 실행 방법

과제를 시작할 때 터미널에서 아래 명령어를 실행해주세요.

```bash
npm run start-assignment
```

작업이 끝날 때까지 터미널을 닫지 않는 것을 권장합니다.

## 기록되는 내용

- 변경된 파일 경로
- 변경된 파일의 간단한 작업 요약
- 변경 순서

## 기록되지 않는 내용

- 디자인 점수
- 합격/탈락 판단
- AI 대화 전문
- 개인 API key 또는 환경 변수
- `node_modules`, 캐시, 임시 파일

## 주의사항

이 기록은 평가자가 작업 흐름을 이해하기 위한 보조 자료입니다. 최종 평가는 결과물, 디자인 방향 설명, AI 활용 기록, 개발 핸드오프, 면접 질문을 함께 보고 판단합니다.

## Figma 사용자 안내

Figma는 클라우드에 저장되므로 `workspace/` 로컬 폴더에 실시간 기록이 생기지 않습니다.
작업 완료 후 Figma 파일이나 링크를 `workspace/design/`에 저장하면 recorder가 기록합니다.

로그 이벤트 수가 적은 것은 Figma 사용자에게 불이익이 없습니다.
최종 평가는 결과물, AI 활용 기록, 핸드오프, 면접을 함께 보고 판단합니다.

## Node.js 안내

이 recorder는 Node.js 18 이상이 필요합니다.
[nodejs.org](https://nodejs.org)에서 LTS 버전을 설치 후 실행해주세요.

# Workspace 안내

과제 작업 파일은 가능한 한 이 폴더 안에 저장해주세요.

## 무엇을 어디에 저장하나

| 폴더 | 용도 |
|------|------|
| `design/` | HTML, 이미지 등 로컬 작업 파일 |
| `handoff/` | 개발자 핸드오프 (`README.md`에 직접 작성) |
| `assets/` | 이미지, 아이콘, 참고 자료 등 보조 파일 |
| `submission/` | 초안이나 메모 공간 |

**최종 제출 답변은 루트의 `submission/template.md`를 작성해주세요.**
`workspace/submission/`은 보조 파일이나 초안을 두는 공간입니다.

## 작업 환경별 안내

HTML, AI 디자인 도구 등 어떤 형식으로 작업해도 됩니다.

---

**이 폴더에서 작업하는 경우** (HTML, Claude Code 등 로컬 파일)

파일을 `workspace/design/`에 저장하면 recorder가 자동으로 기록합니다.
결과물 링크 또는 파일 경로는 `submission/template.md` 1번에 기입해주세요.

---

**외부 도구로 작업하는 경우** (Figma, Stitch, Figma Make, v0 등)

작업이 클라우드에서 이루어지므로 recorder 로그가 자동으로 남지 않습니다. 불이익은 없습니다.

결과물 링크(Figma URL 등)는 `submission/template.md` 1번에 기입해주세요.
`recorder/work-log-template.md`를 열어 어떤 툴로 무엇을 했는지 기록해주세요.

## recorder 실행 안내

Node.js 18 이상이 필요합니다.
설치되어 있지 않다면 [nodejs.org](https://nodejs.org)에서 LTS 버전을 설치해주세요.

실행이 어려운 경우, 과제를 완료한 뒤 `submission/template.md`의 5번 항목에 이유를 적어주세요.
